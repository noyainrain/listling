do_update()¶

Subclass API: Perform the database update.

May be overridden by subclass. Called by update(), which takes care of updating (or initializing) micro specific data. The default implementation does nothing.

create_user(data)¶

Subclass API: Create a new user.

data is the JSON data required for User.

May be overridden by subclass. Called by login() for new users, which takes care of storing the object. By default, a standard User is returned.

create_settings()¶

Subclass API: Create and return the app Settings.

id must be set to Settings.

Must be overridden by subclass. Called by update() when initializing the database.
